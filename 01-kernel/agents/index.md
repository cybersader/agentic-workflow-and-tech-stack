Agents are active executors. Each spawns a **separate context**, runs a task, and returns a summary. Contrast with [skills](../skills/) which load INTO existing contexts as passive expertise.

See [skills-vs-agents principle](../principles/03-skills-vs-agents/) for when to use which.
