## Purpose

This is a research knowledge base. The structured files ARE the project deliverable, not post-hoc documentation.

## Key Principle

**The KB IS the project.** Files evolve as understanding deepens.

## Structure

| Folder | Purpose |
|--------|---------|
| `00-overview/` | Vision, goals, success criteria |
| `01-problem/` | Problem definition, constraints |
| `02-research/` | Analysis, competitors, findings |
| `03-architecture/` | Technical designs, solutions |

## Workflow

```
Session 1: Create overview with initial vision
Session 2: Agent reads overview, has full context
Session 3: Add research findings
Session 4: Synthesize into architecture
```

## Conventions

- SCREAMING_SNAKE_CASE for key documents
- Numbered folders for progression
- Frontmatter on every file
- Cross-reference related files

## SEACOW Mapping

- **Capture:** 01-problem/, raw notes
- **Work:** 02-research/, analysis
- **Output:** 03-architecture/, conclusions
