## Quick Actions (If Using This Structure)

- **New project:** Copy `Templates/project-template/` to your work area
- **New input item:** Adapt `Templates/frontmatter/example-input-frontmatter.md`
- **New output item:** Adapt `Templates/frontmatter/example-output-frontmatter.md`

## Remember

This is an EXAMPLE. Your "system" components might be:
- Scattered config files (no dedicated folder)
- `.github/workflows/` (code repo)
- Plugin configurations (Obsidian)
- `_Templates/` (file share)
- Completely different structure

SEACOW asks "what supports your workflow?" not "create a System/ folder."
