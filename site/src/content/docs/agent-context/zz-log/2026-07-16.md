## Session: claudex/fablex arc — backfill + context-window wedge incident

Backfilling the launcher arc that shipped across commits `6a9c9b7…72da93c` without a worklog (the sessions ran out of context before the Stop-hook could be honored — itself a data point for the incident below):

- **claudex** — Claude Code harness on GPT-5.6-Sol via CLIProxyAPI (loopback proxy translating Anthropic API → Codex OAuth). `claudex-proxy` autostarts the daemon; `claudex-login` does the one-time Codex OAuth.
- **fablex** — Fable 5 orchestrator with `CLAUDE_CODE_SUBAGENT_MODEL=gpt-5.6-sol`, so all delegated subagents run on Sol. Same proxy, needs the *Anthropic* login too (see gotcha).
- **Variant matrix** — house scheme extended: `y` (skip-perms), `r` (continue), `u` (opens with the "ultracode" workflow opt-in prompt), combining into `claudexy/claudexry/claudexyu/claudexryu` (+ `claudrexyu` the dinosaur alias) and the fablex equivalents. `ccrym` = floorless manual-delegation mode (unsets the subagent floor so a named model wins).
- `CLAUDE_CODE_ALWAYS_ENABLE_EFFORT=1` baked into both so `/effort` drives Sol reasoning levels.

### Incident: Sol session wedged past its context window

A long claudex session ran a 19-minute workflow-backed code review, then hit **`API Error: 400 Your input exceeds the context window of this model`** — no auto-compact ever fired. Root cause: through the proxy, Claude Code doesn't know the proxied model's real context window; its auto-compact thresholds are calibrated for Anthropic windows (Fable = 1M), so it sails straight past Sol's smaller limit and the upstream rejects the request.

Worse, the wedge is self-sealing: **`/compact` also fails** ("summarization produced empty response") because the compaction request itself must send the oversized context to the same model. A retry loop burned "3h cogitating" against the same wall.

**Two rescue patterns (both work):**

1. **Window-swap** — resume the *same session* on a big-window Anthropic model (`ccry` in the project dir → Fable/Opus 1M fits it → `/compact` there → exit → `claudexry` hands the now-compacted session back to Sol). The session file is local; which model resumes it is just a flag.
2. **pconv salvage** — `pconv dump <uuid>` the wedged session to markdown, start fresh `claudexy`, feed it only the *tail* (feeding the full dump re-wedges instantly). The completed workflow's findings survive on disk regardless (session jsonl + workflow journal).

User resolved today's instance with a session reset; both recipes documented in the pattern doc.

### Gotcha: the proxy needs BOTH logins

`fablexryu` failed with `502 unknown provider for model claude-fable-5` — the proxy had the Codex OAuth but no Anthropic credential, so it had nowhere to route `claude-*` models. Fix: one-time `cli-proxy-api -config ~/.cli-proxy-api/config.yaml -claude-login`, then restart the proxy (`pkill -f cli-proxy-api`; next launcher call autostarts it). Also field-noted: the "claude.ai connectors are disabled" warning and the "API Usage Billing" banner text in proxy sessions are cosmetic — Claude Code only sees a token; actual metering follows the proxy's OAuth.
