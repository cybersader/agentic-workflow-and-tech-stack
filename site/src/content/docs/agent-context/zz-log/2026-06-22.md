## Session: ShareX text/clipboard "never Pastebin" gotcha added to the image-paste pattern

While diagnosing an unrelated WSL issue, found that the clip2path/ShareX setup on this machine routed **text and clipboard uploads to public Pastebin** even though images go to the Tailscale-gated Zipline. ShareX has a *separate* text uploader from its image uploader, and Pastebin is its out-of-box text default. The exposure is silent: any "Upload text" action, or a `ClipboardUpload` hotkey fired while the clipboard holds text rather than an image, would have published to Pastebin.

The non-obvious part — and the reason this is worth a pattern-doc gotcha rather than a one-off fix — is that **per-hotkey destinations override the global default**. Hotkeys with `"UseDefaultDestinations": false` (which the clip2path clipboard hotkeys use) carry their own `"TextDestination"`, so fixing only the global `ApplicationConfig.json` default still leaks through those hotkeys. There is also no per-service "disable Pastebin" toggle in ShareX; you control exposure by which destination is *selected*, and the only global kill-switch ("Disable upload") disables Zipline too.

Live config on this machine was fixed (global default + all 12 hotkey `TextDestination`s repointed to `CustomTextUploader`; the Zipline custom uploader's `DestinationType` extended with `TextUploader`; verified zero `Pastebin` destinations remain). ShareX rewrites its JSON on exit, so the edits were done with ShareX force-killed, then relaunched.

Shipped:

- `02-stack/patterns/image-paste-pipeline.md` — new "Gotcha: text & clipboard uploads default to public Pastebin" section after Security notes. Documents the separate-text-uploader trap, the no-disable-switch reality, the three config files that must agree (`ApplicationConfig.json` / `UploadersConfig.json` / `HotkeysConfig.json`), the per-hotkey-override pitfall, the close-before-editing requirement, and a one-line verify grep. Tied back to the agentic-workflow rationale: clip2path's value is that agents read from private Tailscale-gated URLs, and a text upload landing on Pastebin breaks that for text the way a public Zipline would for images.
