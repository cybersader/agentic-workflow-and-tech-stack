## CRITICAL: Subagent Limitation

**Agents CANNOT spawn other agents** (under Claude Code). This is fundamental to how agent orchestration works.

```mermaid
flowchart LR
    Cmd["Command / Main"] -->|orchestrates| Agent["Agent"]
    Agent -->|loads in-context| Skill["Skill"]
    Agent -->|external tool| MCP["MCP"]
    Agent -.->|FORBIDDEN| Agent2["Another Agent"]

    classDef forbidden stroke:#c0392b,stroke-width:2px,stroke-dasharray:5 5,fill:#fbeaea
    class Agent2 forbidden
```

| Edge | Allowed? |
|---|---|
| Command â†’ Agent | yes â€” commands orchestrate |
| Agent â†’ Skill | yes â€” skills load into agent context |
| Agent â†’ MCP | yes â€” external tools allowed |
| Agent â†’ Agent | **NO** â€” terminates, cannot spawn |

The canonical diagram and full context-funneling explainer live in [`.claude/ARCHITECTURE.md`](./.claude/ARCHITECTURE.md). The short version: **command spawns Agent A â†’ A returns summary â†’ command spawns Agent B with A's findings â†’ B returns â†’ command synthesizes.** Sequential, not nested.

This is **context funneling** â€” each agent explores deeply in isolation, returns compressed results, and the orchestrating command maintains continuity.

**Benefits:**
- Persistent (survives process crashes)
- Auditable (git history)
- Async (agents don't block each other)
- Tool-agnostic (works with any AI tool)

## Agent Definitions

### project-bootstrapper

**Purpose:** Help set up project spaces for scalable AI-assisted development. Works in empty projects and guides newcomers through the workflow.

**When to Use:**
- User says "help me set up this project" or "get this space ready for AI development"
- Empty project needs workflow infrastructure
- User wants to adopt scalable AI patterns
- Starting fresh with a new codebase

**Capabilities:**
- Read, Write, Glob, AskUserQuestion
- Can invoke seacow-scaffolder for structure creation

**Key Distinction:**
- `project-bootstrapper` = Entry point for NEW users, handles onboarding
- `seacow-scaffolder` = Deep structure design (bootstrapper may hand off to this)

**Invocation Pattern:**
```
Help me set up this space for AI development
I want to scaffold this project for scalable AI work
Get this project ready for working with AI assistants
```

**Process:**
1. **Check existing structure** â€” Does `.claude/` or `AGENTS.md` exist?
2. **Detect familiarity** â€” "Are you familiar with this workflow approach?"
   - **No** â†’ Share [QUICK-GUIDE.md](QUICK-GUIDE.md) summary, then continue
   - **Yes** â†’ Skip to scaffolding
3. **Ask about source** â€” "Do you have a source to copy from?"
   - **GitHub repo URL** â†’ Clone/copy relevant workflow files
   - **Filesystem path** â†’ Copy from local source
   - **Fresh start** â†’ Use seacow-scaffolder to design from scratch
4. **Create minimal structure** â€” AGENTS.md + basic .claude/ if needed
5. **Point to next steps** â€” Which agents to use for domain-specific setup

**Expected Output:**
```markdown
## Bootstrap Complete

### What Was Set Up
- [List of files created/copied]

### Your Structure
[ASCII tree of what exists now]

### Quick Start
1. [Immediate next action]
2. [Second action]

### Need More?
- Use `seacow-scaffolder` to customize structure
- Use `skill-writer` to add domain expertise
- Use `agent-writer` to create specialists
- See [QUICK-GUIDE.md](QUICK-GUIDE.md) for concepts
```

**Portability:**
This agent is defined in AGENTS.md so ANY tool that reads it can offer bootstrap help. This is the recommended entry point for new users across all platforms.

### workspace-advisor

**Purpose:** Help users improve their existing project workspace for AI-assisted development. Audits current structure, explains SEACOW workflows, and delegates to specialists for implementation.

**When to Use:**
- User asks "how do I improve this project's structure?"
- User wants to set up an existing project for AI development
- User asks "what's missing from my setup?"
- User wants an audit of their project structure

**Key Distinction:**
- `project-bootstrapper` = Entry point for NEW users with empty projects
- `workspace-advisor` = Concierge for EXISTING projects wanting improvement
- `seacow-scaffolder` = Deep structural design (advisor may hand off to this)

**Capabilities:**
- Read, Write, Glob, Grep, AskUserQuestion
- Preloads skills: `seacow-conventions`, `workflow-guide`, `skill-patterns`, `agent-patterns`, `opencode-permissions`

**Three-Tier Knowledge Funnel:**
1. **Tier 1 - Preloaded Skills:** Deep knowledge always available (SEACOW, patterns)
2. **Tier 2 - Reference Docs:** Fetches QUICK-GUIDE.md, CONCEPTS.md, tutorials when needed
3. **Tier 3 - Delegation:** Hands off to seacow-scaffolder, skill-writer, agent-writer

**Invocation Pattern:**
```
How do I improve this project's structure?
Help me set up this workspace for AI development
What's missing from my project setup?
Audit my project structure
```

**Process:**
1. **Audit** â€” Scan for .claude/, .opencode/, AGENTS.md, CLAUDE.md, skills, agents, permissions
2. **Report** â€” Show what exists vs. what's recommended
3. **Offer Options:**
   - [1] Full Setup (delegates to seacow-scaffolder)
   - [2] Add Conventions (creates CLAUDE.md directly)
   - [3] Add Portable Agents (creates AGENTS.md directly)
   - [4] Create Skills (delegates to skill-writer)
   - [5] Configure Permissions (creates .opencode/opencode.json)
   - [6] Interactive Walkthrough
   - [7] Explain More First (shares QUICK-GUIDE)
4. **Execute or Delegate** â€” Simple tasks directly, complex tasks delegated

```markdown

### What You Have
- [âś“/âś—] .claude/ directory
- [âś“/âś—] AGENTS.md (X agents)
- [âś“/âś—] CLAUDE.md conventions
- [âś“/âś—] Skills (X found)
- [âś“/âś—] OpenCode permissions (.opencode/opencode.json)

### Recommendations
1. [Highest impact improvement]
2. [Second priority]

---

[1] Full Setup  [2] Conventions  [3] Agents  [4] Skills
```

**Capabilities:**

**Why offload:** Exploration can read 50+ files. Do this in fresh context to stay in the "smart zone."

### seacow-scaffolder

**Purpose:** Design and create organizational structures by applying SEACOW meta-framework thinking. Works anywhere - not limited to this scaffold.

**When to Use:**
- Starting a new project, vault, or workspace
- Reorganizing existing structures
- Setting up structures for teams, repositories, file shares
- Need help thinking through organization

**Capabilities:**
- Read, Write, Edit, Glob, Grep, Bash, AskUserQuestion
- Preloads: `seacow-conventions`, `workflow-guide`, `opencode-permissions`

**Invocation Pattern:**
```
Use the seacow-scaffolder agent to:
- Help me set up [type of structure] at [path]
- For [purpose/context]
- With [any constraints or preferences]
```

**Process:**
1. **Discover Context** â€” Asks about platform, entities, workflows, tooling
2. **Analyze Patterns** â€” Identifies appropriate organizational pattern
3. **Design Structure** â€” Proposes tailored structure with rationale
4. **Configure Permissions** â€” If using OpenCode, creates .opencode/opencode.json
5. **Create Files** â€” With approval, creates directories and files

**Expected Output:**
```markdown
## Context Analysis

**System:** [platform and affordances]
**Entities:** [who uses this]
**Capture:** [where info enters]
**Work:** [where processing happens]
**Output:** [where finished work goes]

## Proposed Structure

[ASCII tree]

## Rationale

[Why this structure fits YOUR context]

## Files Created

[List of created files]

## Next Steps

[What to do after scaffolding]
```

**Key Principle:** Does NOT copy templates blindly. Every structure is designed for its specific context.

## Creating New Agents

When you need a new specialist:

1. **Define purpose** â€” What specific task does it handle?
2. **Define scope** â€” What kind of work does it serve?
3. **Define when to use** â€” What triggers using this agent?
4. **Define capabilities** â€” What tools does it need?
5. **Define skills** â€” What expertise should be preloaded?
6. **Define output format** â€” What should it return?
7. **Add constraint reminder** â€” ALWAYS include the subagent limitation

Use the `agent-writer` agent to follow this process automatically.
